# Row Notes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add one persistent editable note per project row, including recent/all-note views and backward-compatible JSON v1→v2 loading.

**Architecture:** `RowNote` is embedded in each Codable `StoredProject`; project methods enforce one note per row and blank-as-delete behavior. `JSONProjectStore` exposes note operations and atomically persists archive version 2, while SwiftUI views depend only on Store methods.

**Tech Stack:** Swift 6, SwiftUI, Combine, Codable JSON, Swift Testing, String Catalog, XcodeGen.

## Global Constraints

- One note per row per project.
- Row must be zero or greater.
- User note text must never be translated.
- Blank new notes are ignored; blank edits delete existing notes.
- Existing JSON v1 projects must load without data loss.
- Newly saved archives use version 2.
- Keep Apple Watch and iCloud outside this phase.
