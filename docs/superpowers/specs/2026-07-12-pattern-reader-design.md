# KnitNote 圖解閱讀設計規格

日期：2026-07-12

## 目標

讓每件作品可關聯多份圖片或 PDF 圖解，並在 App 內縮放、翻頁及使用高亮線追蹤閱讀位置。匯入後的檔案由 App 自行保存，不依賴原始檔案位置。

## 支援格式與限制

- PDF、PNG、JPEG、HEIC。
- 單檔上限 100 MB；超過時拒絕匯入並顯示本地化說明。
- 第一版只從系統檔案選擇器匯入，不包含相機拍攝或照片圖庫選取。
- 損毀、空白或不支援的檔案不建立紀錄。

## 資料模型

新增 `PatternDocument`：

- `id: UUID`
- `displayName: String`
- `kind: image | pdf`
- `storedFilename: String`
- `createdAt: Date`
- `lastOpenedAt: Date?`
- `pageIndex: Int`，圖片固定為零。
- `zoomScale: Double`
- `contentOffsetX/Y: Double`
- `highlightEnabled: Bool`
- `highlightPosition: Double`，使用 0...1 的相對位置。

`StoredProject` 新增 `patterns: [PatternDocument]`。每份圖解 ID 與內部檔名唯一；顯示名稱可以重複。

## JSON v3 遷移

- 新寫入的 Archive 版本為 3。
- v1、v2 缺少 `patterns` 時解碼為空陣列。
- 遷移不得改變作品、排數、筆記或既有日期。
- 第一次資料變更時以 v3 原子寫回。

## 檔案保存

- 匯入時先取得安全範圍存取權，再驗證類型與大小。
- 複製至 Application Support 下的 `KnitNote/Patterns/<project-id>/`。
- 內部檔名使用圖解 UUID 加原副檔名，避免同名覆蓋。
- 檔案複製成功後才建立 JSON 紀錄。
- 刪除圖解時先刪除紀錄並保存；檔案刪除失敗需記錄並在後續清理，不讓幽靈紀錄留在 UI。

## 作品頁流程

- 沒有圖解時顯示「加入圖解」。
- 有圖解時顯示最近開啟的一份、加入按鈕及「查看全部」。
- 全部圖解依最近開啟時間、建立時間由新到舊排序。
- 支援重新命名與刪除確認。
- 點擊圖解更新 `lastOpenedAt` 並開啟閱讀器。

## 閱讀器

### 圖片

- 保持比例顯示。
- 支援雙指縮放、拖移與雙擊回到適合畫面大小。
- 離開時保存縮放與相對偏移。

### PDF

- 使用 PDFKit 顯示多頁 PDF。
- 支援翻頁、連續捲動與縮放。
- 顯示目前頁碼與總頁數。
- 離開時保存目前頁碼、縮放與相對偏移。

### 高亮線

- 半透明橫向色帶疊加於閱讀內容上方。
- 可開關並上下拖移。
- 位置以閱讀區高度比例保存，旋轉或換裝置後仍接近原位置。
- 每份圖解分別保存狀態。

## 錯誤處理

- 找不到保存檔案時顯示錯誤，提供關閉與移除失效紀錄。
- PDF 無頁面、圖片無法解碼或檔案過大時拒絕匯入。
- JSON 保存失敗時移除剛複製的檔案並保留原資料狀態。
- 重新命名只修改顯示名稱，不改內部檔名。
- 使用者內容不因語言切換而翻譯。

## 本地化與無障礙

- 匯入、閱讀、頁碼、縮放重設、高亮、改名、刪除及錯誤訊息支援英文與繁體中文。
- VoiceOver 提供檔名、類型、頁碼及按鈕標籤。
- iPad 支援大畫面閱讀；iPhone、iPad、Mac 共用功能行為。

## 測試

- v1/v2 JSON 載入後取得空圖解集合。
- v3 往返保存不遺失閱讀狀態。
- 格式、大小、空 PDF 與損毀圖片驗證。
- 同名匯入產生不同內部檔名。
- 複製失敗不建立紀錄。
- 刪除、改名與最近開啟排序。
- 高亮位置限制於 0...1，頁碼限制於有效範圍。
- 英文／繁中字串完整。
- macOS 與 iOS 建置通過；原有測試不退步。

## 驗收標準

- 每件作品可匯入、閱讀、改名及刪除多份圖片與 PDF。
- App 重開後圖解及閱讀位置仍存在。
- PDF 頁碼、圖片/PDF 縮放與高亮位置可恢復。
- 舊作品自動相容且資料不遺失。
- iPhone、iPad 與 Mac 均可操作。

## 非目標

- 圖片標註、文字辨識、AI 解讀或圖解編輯。
- 相機、照片圖庫、網頁下載或分享擴充。
- Apple Watch 圖解。
- iCloud 檔案同步。
