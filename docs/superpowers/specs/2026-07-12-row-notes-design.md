# KnitNote 每排筆記設計規格

日期：2026-07-12

## 目標

讓使用者在作品的特定排數保存一則文字筆記，記錄針法變更、錯誤、材料狀況或下次繼續事項。筆記與作品一起本機保存，App 重開及介面語言切換後保持不變。

## 資料模型

新增 `RowNote`：

- `row: Int`：對應排數，必須大於或等於零，且在同一作品內唯一。
- `text: String`：使用者原始文字；保存前只移除頭尾空白。
- `createdAt: Date`：首次建立時間。
- `updatedAt: Date`：最後修改時間。

`StoredProject` 新增 `rowNotes: [RowNote]`。同一排再次保存會更新既有筆記，不建立第二則。空白文字不建立新筆記；把既有筆記改成空白等同刪除。

## JSON 相容性

- 新保存的 `ProjectArchive.version` 為 `2`。
- v2 作品包含 `rowNotes`。
- 解碼舊 v1 JSON 時，缺少 `rowNotes` 必須自動使用空陣列。
- 載入後第一次變更資料時，以 v2 格式原子寫回。
- 不修改原有作品 ID、名稱、排數及日期。

## 使用流程

### 目前排數筆記

- 作品計數頁新增「筆記」按鈕。
- 點擊後開啟目前排數的筆記編輯畫面。
- 若該排已有筆記，文字欄預先載入原內容。
- 儲存後留在同一作品，並更新作品的 `updatedAt`。
- 取消不變更任何資料。

### 最近筆記

- 作品頁在計數控制下方顯示最近三則筆記。
- 依排數由高至低排列；相同排數不可能重複。
- 每列顯示排數、內容摘要及最後修改時間。
- 有超過三則時顯示「查看全部」。

### 全部筆記

- 顯示作品的所有筆記，依排數由高至低排列。
- 點擊一則筆記可編輯。
- 滑動刪除時先顯示包含排數的確認視窗。
- 確認後刪除；取消不變更資料。

## 架構

- 筆記規則放在 `StoredProject` 的純 Swift 方法中。
- `JSONProjectStore` 提供保存、刪除與查詢筆記的方法，畫面不直接改寫陣列。
- UI 只依賴 Store 公開介面，未來更換 SwiftData／CloudKit 時不重寫畫面流程。
- 使用既有原子 JSON 寫入，任何筆記變更完成後立即持久化。

## 本地化與無障礙

- 按鈕、標題、空白狀態、排數標籤、刪除確認與錯誤訊息支援英文及繁體中文。
- 使用者筆記不翻譯。
- VoiceOver 朗讀排數、筆記摘要與操作按鈕。
- 編輯欄支援動態字級與多行輸入。

## 錯誤處理

- 儲存失敗時保留編輯文字並顯示本地化錯誤。
- 找不到指定作品時關閉編輯畫面，不建立孤立筆記。
- 排數不得為負數。
- JSON 損毀時維持既有 Store 行為：不覆寫損毀檔案，並讓上層顯示可處理的載入狀態；本階段不建立自動修復工具。

## 測試

- 同排新增後再次保存只保留一則且更新時間正確。
- 空白新筆記不保存，清空既有筆記會刪除。
- 不同排數可各自保存筆記並正確排序。
- JSON v2 往返保存不遺失筆記。
- v1 JSON 可載入且得到空筆記集合。
- 改名、排數與日期在 v1→v2 遷移中保持一致。
- 刪除筆記後重建 Store，筆記仍保持刪除。
- 英文與繁中字串完整。
- macOS 與 iOS 建置通過，原有七項測試不退步。

## 驗收標準

- 使用者可為目前排數新增、查看、修改及刪除一則筆記。
- 最近三則與全部筆記排序正確。
- App 重開後筆記仍存在。
- v1 現有作品無需手動操作即可繼續使用。
- 語言切換不改變筆記。
- iPhone、iPad 與 Mac 介面可用且沒有文字截斷。

## 非目標

- 同一排多則筆記。
- 語音筆記、圖片、附件或 AI 整理。
- 搜尋、標籤或跨作品筆記彙整。
- Apple Watch 筆記。
- iCloud 同步。
