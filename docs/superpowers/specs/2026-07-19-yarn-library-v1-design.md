# 毛線庫基本版設計

日期：2026-07-19

## 目標

為 KnitNote 建立簡單、可持續擴充的毛線庫。使用者能快速新增毛線、記錄庫存、查看照片並將同一筆毛線連結到多個作品，同時保持現有 App 的水彩配色與低操作負擔。

## 第一版範圍

- 新增、查看、編輯及刪除毛線。
- 使用相機拍攝或從相簿選擇毛線照片。
- 記錄球數與剩餘重量；兩者可只填一項或同時填寫。
- 將一筆毛線連結到多個作品。
- 支援繁體中文與英文。
- 支援 iPhone 與 iPad，沿用現有安全範圍和水彩主題。

第一版不包含搜尋、篩選、低庫存提醒、自動扣除用量或材料缺口估算。

## 資料模型

每筆毛線包含：

- 唯一識別碼。
- 毛線名稱，唯一必填欄位。
- 照片檔名，可省略。
- 品牌與系列。
- 顏色、色號與染缸號。
- 剩餘球數，可為非負小數。
- 剩餘重量，可為非負小數並以克保存。
- 保存位置。
- 備註。
- 已連結作品識別碼集合。
- 建立日期與修改日期。

同品牌、同系列的不同顏色各自保存為獨立毛線。空白球數或重量代表尚未記錄，不等同於零。

## 保存架構

毛線資料整合進目前的 JSON 資料庫，與作品一同保存。資料封存格式新增可省略的毛線集合；讀取舊版資料時，缺少集合即視為空毛線庫。第一次寫入後以新版格式保存，既有作品、計數器、圖解與筆記不可遺失。

照片沿用目前作品照片的檔案保存原則：驗證影像內容、產生唯一檔名、替換成功後才刪除舊檔。刪除毛線時一併刪除照片；照片操作失敗不得破壞已提交資料。

## 毛線庫首頁

首頁採用兩欄照片格狀版面：

- 右上角提供新增按鈕。
- 每張卡片顯示照片、完整毛線名稱、顏色與剩餘球數。
- 若球數未記錄但有重量，卡片改顯示剩餘重量。
- 若兩者皆未記錄，不顯示虛構的零庫存。
- 沒有照片時顯示符合現有美術風格的預設毛線圖示。
- 點卡片進入詳細頁；長按提供編輯與刪除。
- 空毛線庫顯示 Lemon 與簡短新增提示。

iPad 使用相同資訊階層並增加欄數，但不為 iPad 建立另一套操作流程。

## 新增與編輯

新增頁只要求毛線名稱。照片、品牌、系列、顏色、色號、染缸號、球數、重量、保存位置、備註與作品連結皆可略過，之後再補填。

照片來源同時支援相機與相簿。使用者按導覽列的「完成」即驗證並保存，不增加另一個儲存按鈕。名稱去除首尾空白後不可為空；球數與重量不可為負數。驗證失敗時保留已輸入內容並在相關欄位附近顯示清楚提示。

## 詳細頁與作品連結

詳細頁顯示照片、庫存、品牌系列、顏色色號、染缸號、保存位置、備註及已連結作品。右上角鉛筆按鈕開啟編輯頁。

一筆毛線可連結多個作品。點選已連結作品可直接進入作品頁。作品標記完成後仍保留連結。刪除作品時，只清除所有毛線中的該作品識別碼，不刪除毛線。刪除毛線不修改或刪除作品資料，且必須先顯示確認提示。

第一版不依作品進度自動扣除球數或重量；庫存只在使用者明確編輯後變更。

## 本地化與無障礙

所有標題、欄位、按鈕、錯誤訊息、刪除確認、空狀態及無障礙標籤均加入字串目錄，第一版提供繁體中文與英文。數值依使用者地區格式顯示，但重量在資料層統一以克保存。

圖片必須有可理解的無障礙描述；卡片的朗讀內容包含毛線名稱、顏色及已記錄的庫存。互動控制維持至少 44×44 點觸控範圍，並在 iPhone 與 iPad 安全範圍內顯示。

## 錯誤與資料一致性

- 不接受負數球數或重量。
- 無效照片不得覆蓋已保存照片。
- 保存失敗時保留編輯畫面及輸入值，並顯示可重試訊息。
- 刪除毛線前確認；刪除完成後清理照片。
- 讀取資料時忽略已不存在的作品連結，下一次保存時清理無效識別碼。
- 毛線資料不得改變完成作品的計數器鎖定狀態。

## 驗收與測試

核心測試涵蓋：

- 新毛線預設值、名稱正規化及非負庫存驗證。
- 球數與重量分別保存、同時保存及空白值往返。
- 一筆毛線連結多個作品。
- 刪除作品只清除連結，刪除毛線不改變作品。
- 舊版 JSON 讀取為空毛線庫，以及新版資料完整往返。
- 毛線新增、編輯、刪除及照片替換的持久化。
- 照片驗證、唯一檔名與失敗回復。
- 繁中與英文介面字串完整性。
- 毛線首頁、詳細頁、編輯頁和作品連結的畫面契約。

完成前需執行全部 Swift 測試，建置 iOS 模擬器版本，並在 iPhone 與 iPad 檢查格狀卡片、長名稱、大字模式、相機／相簿、鍵盤輸入及安全範圍。

## 後續版本

基本版穩定後，可依序加入搜尋與篩選、低庫存提醒、材料需求比較、作品用量扣除、毛線替代建議及跨裝置同步。這些功能不得增加第一版新增毛線的必填欄位。
