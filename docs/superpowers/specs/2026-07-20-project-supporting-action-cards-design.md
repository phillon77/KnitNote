# 作品筆記／圖解卡片設計

日期：2026-07-20

## 目標

把作品詳情頁目前並排的圓形「筆記」與「圖解」按鈕，改成與「編織計算工具」一致的全寬水彩白色卡片，讓名稱清楚、點擊範圍更大，並維持畫面風格一致。

## 畫面

- 「編織計算工具」下方依序顯示「筆記」與「圖解」。
- 每項各自使用一張 `WatercolorCard`。
- 卡片內顯示原有圖示及完整本地化名稱，內容靠左，整列可點擊。
- 保留作品頁既有 620 點最大寬度、垂直間距、Dynamic Type 與 VoiceOver 標籤。
- 不新增副標題、箭頭、徽章或其他裝飾。

## 行為

- 「筆記」仍開啟目前選取計數器與目前段數的筆記頁。
- 「圖解」仍開啟此作品的圖解頁。
- 不修改完成作品的既有權限、資料或導覽方式。
- 不新增或修改多語系 key，沿用 `notes.edit` 與 `patterns.open`。

## 驗證

- 原本的並排 `HStack` 與 icon-only `supportingButton` 不再用於這兩個入口。
- 兩項操作皆位於獨立 `WatercolorCard`，使用完整 `Label` 並全寬靠左。
- 「編織計算工具 → 筆記 → 圖解 → 編織日記」順序固定。
- 完整 Swift 測試、macOS build 與差異完整性檢查通過。
