# 織圖優先與 iPad 閱讀器調整設計

## 目標

依實際編織使用情境，提高織圖在作品頁中的優先順序，縮小高亮範圍，並修正 iPad 橫向字體過小與直向分頁控制遮住內容的問題。既有 iPhone 閱讀行為、頁碼儲存、高亮位置、手寫標記與每頁筆記必須維持穩定。

## 名稱與在地化

- 繁體中文介面中所有面向使用者的「圖解」統一改為「織圖」。
- 範圍包含底部分頁、作品頁入口、織圖清單、新增與刪除提示、錯誤訊息、空白狀態、閱讀器工具名稱及 VoiceOver 文案。
- 英文維持既有 `Pattern`／`Patterns` 用語。
- 不重新命名程式內部的 `PatternDocument`、儲存欄位、檔案目錄或 localization key，避免資料遷移與相容性風險。

## 作品頁資訊順序

作品詳細頁的主要內容依序為：

1. 作品照片
2. 織圖入口
3. 每排筆記入口與最近筆記
4. 六個計數器
5. 編織工具資料（有資料時顯示）
6. 編織計算工具
7. 編織日記

完成狀態仍緊接在作品照片後顯示，屬於作品狀態而不是主要功能區塊。工具資料與計算工具必須位於計數器後、日記前。

## 閱讀器頂部

- iPhone 與 iPad 的織圖閱讀畫面都取消可見的 PDF／圖片標題。
- 上方導覽區只保留完成、高亮、高亮模式、手寫標記及本頁筆記等操作。
- 織圖名稱仍顯示於進入閱讀器前的織圖清單。
- VoiceOver 仍可取得目前織圖名稱，但名稱不得佔用可見版面高度。

## 高亮樣式

### 橫向高亮

- 保留半透明黃色帶。
- 可見高度由 44 pt 改為 22 pt，以單排文字為主要覆蓋範圍。
- 拖曳手勢的有效高度維持至少 44 pt；視覺高度與觸控高度分離。
- 保留 VoiceOver adjustable action 與既有位置儲存方式。

### 縱向高亮

- 改為粉紅色實心細線，可見寬度為 3 pt。
- 移除目前 44 pt 半透明帶與拖曳圖示。
- 拖曳手勢的有效寬度維持至少 44 pt；視覺寬度與觸控寬度分離。
- 保留 VoiceOver adjustable action 與既有位置儲存方式。

### 交叉模式

交叉模式同時使用 22 pt 黃色半透明橫帶與 3 pt 粉紅實心縱線，各自保留獨立拖曳與位置狀態。

## iPad 自適應閱讀策略

### 橫向

- PDF 初次顯示與裝置旋轉到橫向時，自動採用符合可用寬度的比例，不再以完整 A4 頁面符合高度。
- 可用寬度必須扣除右側六個計數器安全區。
- 同一頁超出垂直可用範圍時，允許使用者上下拖動查看。
- 上一頁與下一頁仍維持單頁切換，不改成連續頁面捲動。
- 旋轉或重算比例不得改變目前頁碼、高亮位置、手寫內容或本頁筆記。

### 直向

- PDF 仍使用目前整頁單頁閱讀策略。
- 「上一頁／頁碼／下一頁」改放在 PDF 容器下方的固定控制列。
- 固定控制列取得自己的版面高度，不使用 overlay，不得覆蓋 PDF 最下方內容。
- PDF 可用高度須扣除控制列高度。

### iPhone

- 保留目前的 PDF 自動縮放、單頁切換與底部分頁控制外觀。
- 本次不改變 iPhone 的 PDF 縮放策略。
- 只套用取消可見標題、在地化名稱與新高亮樣式。

## 版面元件邊界

- `ProjectDetailView` 只負責作品功能區塊的順序。
- `PatternReaderView` 根據裝置類型與可用尺寸選擇控制列配置，並保留右側計數器安全區。
- `PatternReaderControls` 將右側計數器與頁面控制拆成可獨立配置的視覺單元；iPad 直向頁面控制位於閱讀器外，iPhone 維持 overlay。
- `PDFReaderView`／Coordinator 負責將 iPad 橫向的顯示比例設為符合可用寬度；不得修改 `PatternReadingState` 的資料格式。
- `HighlightOverlay` 將可見形狀與 44 pt 拖曳命中區分離。

## 旋轉與狀態處理

- 尺寸變更只觸發顯示比例與可用區域重算。
- PDF 頁面仍由 `PDFPageNavigator` 與既有 page request gate 控制。
- 重算比例不得觸發錯誤的頁面前進、回到第一頁或清空標記。
- 閱讀狀態的儲存時機與現有資料結構保持不變。

## 測試與驗收

### 自動測試

- 作品頁原始碼合約測試確認主要區塊順序。
- localization contract 確認所有相關繁體中文值使用「織圖」，且不殘留面向使用者的「圖解」。
- 高亮版面規則測試確認橫向 22 pt、縱向 3 pt、兩者命中範圍至少 44 pt。
- 閱讀器版面策略測試確認：iPad 橫向符合寬度、iPad 直向使用外部固定分頁列、iPhone 保留原策略。
- 既有頁碼恢復、高亮、手寫與每頁筆記測試必須全部通過。

### 實際畫面驗收

- iPhone 直向：內容與分頁行為不退步，標題空間釋放，高亮可正常拖動。
- iPad 直向：PDF 最下方不被上一頁／下一頁控制遮擋。
- iPad 橫向：A4 織圖以寬度顯示，文字可閱讀，可上下拖動查看同一頁。
- iPad 直向與橫向來回旋轉：頁碼、高亮、手寫標記與本頁筆記保持不變。
- 繁體中文所有主要入口與訊息顯示「織圖」。

## 非目標

- 不重做 PDF 引擎。
- 不改成連續多頁閱讀。
- 不新增使用者自訂高亮粗細。
- 不改變 iPhone 的縮放策略。
- 不變更織圖資料格式或執行資料遷移。
